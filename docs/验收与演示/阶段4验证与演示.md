## 2. 环境前置

- Windows PowerShell、Python 3.12、uv、Node.js/npm 和 Docker Desktop。
- Docker 使用 `pgvector/pgvector:pg16`；完整演示需要 PostgreSQL、migrate、API、Worker 和 gateway 镜像。
- 运行 Docker 重启测试前，可用物理内存至少为 2 GiB；本轮所有重任务均串行执行。

## 9. 失败排查

- `/health` 成功但 `/api/ready` 失败：检查 PostgreSQL、Alembic head 和 gateway 到 API 的内部网络；`/health` 不代表数据库就绪。
- Worker unhealthy：先检查 PostgreSQL，再执行 `python -m app.worker.health`；恢复验收必须看到晚于数据库重启完成时间的新心跳。
- 集成测试出现表不存在：一次性基础库尚未执行 `uv run alembic upgrade head`。
- 集成测试出现测试用户外键残留：确认使用当前 `tests/database_cleanup.py`，不要在失败后复用已经污染的测试库。
- SSE 中断或超时：确认 Nginx `proxy_buffering off`、长读取超时和 gateway `/api` 代理。
- Docker 构建前若物理可用内存低于 2 GiB，停止本轮重任务；不要并行构建 API、Worker 和 gateway。
- 构建报告显示大 chunk：当前不阻塞桌面演示，阶段 5 再做按路由拆包和手机性能验收。
