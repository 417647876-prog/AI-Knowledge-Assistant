# 阶段 2C：文档管理验证与演示

本文用于验证阶段 2C 的历史文档加载、五种处理状态、失败重处理、安全删除和删除后的 RAG 引用清理。所有命令默认面向本机开发环境，不适用于共享数据库或生产环境。

## 1. 本阶段能力

- 进入或切换知识库时，从数据库重新加载历史文档。
- 显示 `pending`、`parsing`、`embedding`、`ready`、`failed` 五种后端状态。
- 页面重新打开后，自动恢复处理中任务的状态轮询。
- `ready` 和 `failed` 文档可以重新处理或删除。
- 处理中任务禁止重复重处理和删除。
- 删除文档时级联清理入库任务、切片和向量，并删除本地原文件。
- 删除后再次提问，不再引用已删除文档。

## 2. 环境准备与数据库迁移

从仓库根目录打开 PowerShell：

```powershell
Set-Location (git rev-parse --show-toplevel)
docker compose -f deploy/docker-compose.yml up -d
Set-Location backend
$env:APP_ENV = "development"
uv sync --dev
uv run alembic upgrade head
uv run alembic current
```

`alembic current` 应显示 `20260714_04`。该迁移为 `ingestion_jobs` 增加非空 `created_at`，并建立 `(document_id, created_at)` 索引。旧任务会优先用 `started_at` 回填，其次使用 `finished_at`，两者都没有时使用迁移时间。

执行迁移前必须确认连接的是本机开发数据库。不要对共享库或生产库直接照抄命令。

## 5. 浏览器演示步骤

1. 登录普通用户，创建一个测试知识库。
2. 上传 `backend/tests/fixtures/documents/01-年假制度.txt`。
3. 观察状态依次经过“等待处理”“解析中”“向量化中”，最后变为“可用”。阶段切换较快时，页面可能只观察到其中部分中间状态。
4. 刷新页面，确认文档仍在列表中。
5. 点击“重新处理”，确认当前行进入处理中，重新处理和删除按钮暂时禁用，最终回到“可用”。
6. 提问“员工入职满一年后有多少天年假？”，确认回答包含该文件的引用。
7. 点击“删除”，先取消一次，确认文档仍存在；再次操作并确认删除，确认文档行消失。
8. 再次提出相同问题，确认引用中不再出现已删除文档。

删除不可恢复。删除成功会清理该文档的任务、切片、向量和本地原文件。

## 6. 权限 404 验收

1. 使用用户 Alice 创建知识库并上传文档，记录知识库 ID 和文档 ID，不记录密码或令牌。
2. 使用另一个普通用户 Bob 登录。
3. Bob 调用 Alice 的文档列表、单文档查询、重新处理和删除接口。
4. 确认知识库接口返回 `404 KNOWLEDGE_BASE_NOT_FOUND`，文档接口返回 `404 DOCUMENT_NOT_FOUND`。
5. 确认响应不泄露文件保存路径、资源所有者、数据库连接或令牌。

管理员仍遵循阶段 2B 的全局资源访问规则。

## 7. 删除失败与并发保护

- 文档存在 `pending` 或 `running` 入库任务时，删除返回 `409 DOCUMENT_PROCESSING`。
- 本地文件删除失败时，接口返回 `500 DOCUMENT_DELETE_FAILED`，数据库删除回滚，响应不包含本地路径。
- 文件本来已经不存在时按清理完成处理，数据库记录仍可正常删除。
- 数据库事务和 Windows 文件系统无法组成真正的原子事务；极低概率下，文件删除成功后数据库提交仍可能失败。这一补偿机制不属于阶段 2C。

## 8. 已知边界

- 本阶段没有分页、搜索、批量删除、回收站和删除恢复。
- 本阶段不支持强制取消正在运行的入库任务。
- 状态更新继续使用轮询，不使用 WebSocket 或服务端推送。
- 浏览器手工验收需要本地有效账号和运行中的前后端，自动化测试不会代替账号权限与页面操作的最终人工确认。
