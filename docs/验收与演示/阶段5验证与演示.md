## 2. Funnel 官方规则核对

2026-07-22 重新核对 Tailscale 官方文档后采用以下规则：

- Funnel 仍为 beta，所有套餐可用，客户端最低版本为 1.38.3。
- 必须启用 MagicDNS、HTTPS 证书和允许使用 Funnel 的 `funnel` node attribute；首次命令可能要求在浏览器授权。
- 只允许公网 HTTPS 端口 `443`、`8443`、`10000`，仅 TLS，不直接发布本机数据库或应用内部端口。
- `--bg` 配置会跨 Tailscale/设备重启恢复；关闭时使用与启动相同的 flags 并追加 `off`。
- `reset` 会清除当前节点 Funnel 配置，本项目不使用它；公共 DNS 最长可能需要约 10 分钟传播，且流量有不可配置的带宽限制。

官方入口：

- <https://tailscale.com/docs/features/tailscale-funnel>
- <https://tailscale.com/docs/reference/tailscale-cli/funnel>

## 3. 环境和安全前置

1. 可用物理内存至少为 2 GiB；低于门槛时只阅读和轻量编辑，不执行测试、构建或容器重任务。
2. 完整 Compose 的 gateway 必须在 `127.0.0.1:8080` healthy，`/api/ready` 返回 HTTP 200。
3. `deploy/.env` 只保存在本机并使用随机密钥；公网 HTTPS 的 `TRUSTED_ORIGINS` 必须精确填写 Funnel 地址，Cookie 使用 secure。
4. Tailscale CLI 已安装、登录且 `BackendState=Running`，首次 Funnel 已由用户在浏览器批准。
5. 公网只允许 Funnel 代理 `http://127.0.0.1:8080`。PostgreSQL、API、Worker、Vite 和内部指标均不得发布。

## 4. 显式启动和停止 Funnel

先启动并检查完整 Compose，再启用 Funnel：

```powershell
Set-Location (git rev-parse --show-toplevel)
docker compose -f deploy/docker-compose.yml up -d
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:8080/api/ready
powershell.exe -NoProfile -ExecutionPolicy Bypass -File deploy/start-funnel.ps1 -ConfirmEnable
```

脚本会拒绝低内存、缺失 CLI、未登录、gateway 未就绪、旧项目标记或任意已有 Funnel 配置；成功时只输出公网 HTTPS 地址，不输出密码、Token 或 Key。首次授权弹出浏览器后，完成授权并重跑命令。

演示结束只关闭本项目映射，不停止 Docker、不删除卷：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File deploy/stop-funnel.ps1 -ConfirmDisable
docker compose -f deploy/docker-compose.yml down
```

停止脚本要求本项目的 `deploy/.stage5-funnel.json` 标记与当前目标一致。该文件被 Git 忽略；若不一致则拒绝操作，禁止改用 `tailscale funnel reset` 绕过 owner 边界。

## 5. PostgreSQL 与 uploads 备份恢复

```powershell
Set-Location (git rev-parse --show-toplevel)
powershell.exe -NoProfile -ExecutionPolicy Bypass -File deploy/backup.ps1 `
  -DestinationRoot D:\AI-Knowledge-Backups -UseDocker
powershell.exe -NoProfile -ExecutionPolicy Bypass -File deploy/restore.ps1 `
  -BackupDirectory D:\AI-Knowledge-Backups\stage5-backup-YYYYMMDD-HHMMSS `
  -UseDocker -ConfirmRestore
```

备份目录包含 `manifest.json`、`database.dump` 和 `uploads/`。不包含 `.env`、JWT、gateway secret、模型 Key、镜像、日志和 Hugging Face 缓存。恢复会短暂停止 API、Worker、gateway，但不执行 `down -v`，不删除持久卷。

## 6. 真实手机移动网络十步验收

以下步骤必须在手机关闭 Wi-Fi、未连接同一局域网的条件下，通过脚本输出的 HTTPS 地址执行；账号、知识库名、文件名、问题和回答正文只在现场查看，不写入报告。

1. 使用账号 A 登录，确认刷新页面后仍能恢复认证状态。
2. 新建知识库，上传小型 TXT，观察 queued/processing/ready 的 Worker 状态闭环。
3. 发起流式问答，确认增量内容、完成状态和可追溯引用。
4. 打开引用底部抽屉，确认来源和快照可读，页面无横向滚动。
5. 进入“我的”，确认额度、上海时区重置点、用量和未知 Token/费用展示正确；创建并删除自己的反馈。
6. 删除文档或知识库，在回收站确认不可检索、可恢复，随后恢复。
7. 使用账号 B 登录，确认无法看到账号 A 的知识库、会话、反馈和回收站；安全 404/空列表均不得泄露存在性。
8. 回到账号 A，流式回答中断开移动网络，再恢复网络；状态应为 interrupted，只允许手动重新生成，不自动续传。
9. 触发手动重新生成，确认旧 SSE 不覆盖新回答，引用仍对应新消息。
10. 使用低额度账号触发问答或上传拒绝，确认在外部模型调用前返回稳定拒绝，管理员页面只显示脱敏运营元数据。

完成后从手机网络额外确认：公网只能访问页面、gateway 健康和 `/api`；数据库、Worker、Uvicorn、Vite、内部指标无公网端口。随后执行 stop 脚本并确认 HTTPS 地址失效。
