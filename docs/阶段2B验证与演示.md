# 阶段 2B：认证、权限隔离与本地验收

本文用于验证阶段 2B 的账号认证、角色权限、知识库隔离和窄屏可用性。所有命令默认只面向当前项目的本机开发环境，不适用于共享库、远程库或生产库。

## 1. 安全边界

- 系统没有公开注册；首个管理员由命令行创建，后续账号由管理员页面创建。
- Access Token 只保存在前端内存中；Refresh Token 使用 HttpOnly Cookie，数据库只保存其哈希。
- `/health`、`/ready` 和 OpenAPI 匿名可用；知识库、文档、问答与用户管理接口均需要认证。
- 普通用户访问其他用户资源返回 404，管理员可以查看和操作全部知识库。
- 本地重置脚本会对 `knowledge_bases` 执行级联清理，文档、切片和任务也会被删除。它不删除用户，但只能在明确确认目标后运行。

## 2. 环境准备

```powershell
docker compose -f deploy/docker-compose.yml up -d
Set-Location backend
uv sync --dev
Copy-Item .env.example .env   # 仅首次需要；已有本地 .env 时不要覆盖
```

开发环境可以使用示例 JWT 密钥。生产环境绝不能使用默认密钥，必须提供随机高强度 `JWT_SECRET_KEY`，设置 `REFRESH_COOKIE_SECURE=true`，并把 `TRUSTED_ORIGINS` 限制为实际 HTTPS 站点。

## 3. 显式重置、迁移与首个管理员

### 3.1 重置前核对

先查看最终生效配置，不要打印密码部分。目标必须同时满足：

- `APP_ENV=development`；
- 数据库主机是 `localhost`、`127.0.0.1` 或 `::1`；
- 数据库名是当前项目本地开发库 `knowledge`；
- PostgreSQL 端口是预期的本机 `5432`；
- 没有通过 URL 查询参数覆盖 host、port、dbname、user 或 password。

如果任何一项不明确，停止操作。不要对远程、共享或 production 数据库运行重置。

### 3.2 安全执行顺序

```powershell
Set-Location backend
$env:APP_ENV = "development"
uv run python -m scripts.reset_development_data --yes
uv run alembic upgrade head
uv run python -m scripts.create_admin --username admin
uv run uvicorn app.main:app --host 127.0.0.1 --port 8000 --loop app.core.event_loop:new_event_loop
```

`reset_development_data --yes` 仍会在执行前输出非敏感目标信息并由脚本再次校验。`create_admin` 默认使用不回显的交互式密码输入。若自动化场景必须使用环境变量，只在当前 PowerShell 进程临时设置 `INITIAL_ADMIN_PASSWORD`，创建后立即删除；不要把明文密码写进命令历史、文件或报告。

另开终端启动前端：

```powershell
Set-Location frontend
npm.cmd install
npm.cmd run dev -- --host 127.0.0.1 --port 5173
```

## 4. 认证版 smoke

后端使用 Fake Provider 可避免下载模型和调用付费服务：

```powershell
$env:EMBEDDING_PROVIDER = "fake"
$env:CHAT_PROVIDER = "fake"
$env:RAG_SCORE_THRESHOLD = "-1"
uv run uvicorn app.main:app --host 127.0.0.1 --port 8000 --loop app.core.event_loop:new_event_loop
```

在另一个后端终端设置测试账号。下面的转换只把密码放入当前进程环境；执行完立即清除：

```powershell
$env:SMOKE_USERNAME = "admin"
$securePassword = Read-Host "冒烟测试账号密码" -AsSecureString
$env:SMOKE_PASSWORD = [System.Net.NetworkCredential]::new("", $securePassword).Password
uv run python -m scripts.smoke_test
Remove-Item Env:SMOKE_PASSWORD
Remove-Variable securePassword
```

脚本顺序固定为 `health → ready → login → me → create knowledge base → upload → poll → question → logout`。通过时返回 0；失败时返回非 0，并给出脱敏的 HTTP 状态码、错误码和 request ID。脚本不输出密码、Access Token 或 Refresh Token。

## 5. 真实权限验收

使用真实 Edge 打开 <http://127.0.0.1:5173>，密码和令牌不得写入截图、报告或浏览器自动化日志。

1. 匿名访问 `/health`、`/ready`、`/docs`，确认仍公开；匿名访问 `/api/v1/knowledge-bases`，确认 401。
2. 管理员登录，在“用户管理”创建普通用户 Alice 和 Bob。
3. Alice 登录，创建知识库，上传 `backend/tests/fixtures/documents/01-年假制度.txt`，等待状态为“处理完成”，提问“员工入职满一年后有多少天年假？”，确认回答可接受且引用指向上传文件。
4. 刷新页面，确认 Alice 会话由 Refresh Cookie 恢复。
5. Bob 登录，确认列表中没有 Alice 的知识库；直接访问 Alice 的文档或向 Alice 知识库提问均应返回 404。
6. 管理员重新登录，确认知识库列表显示所有者，并可看到、上传或提问 Alice 的资源。
7. 保留一个 Alice 已打开的会话，再由管理员停用 Alice。Alice 的现有 Access Token 后续请求应立即 401；刷新会话最终失效，回到登录页，且不能再次登录。
8. 管理员刷新页面，确认会话保持；点击退出后回到 `/login`，刷新和直接进入受保护页都不能恢复会话。

记录状态码、错误码和 request ID 时只记录非敏感值。验收账号可留在本地开发库中，但交付不能依赖知道这些账号的密码。

## 6. 320px 窄屏验收

Edge 开发者工具设置 `320 × 800`，分别检查登录页、工作台和管理员用户页：

- 页面根节点满足 `document.documentElement.scrollWidth <= document.documentElement.clientWidth`；
- 登录按钮、创建知识库、上传、提问、创建用户、启停和退出等关键操作可达；
- 管理员表格可在局部容器内滚动或切换为卡片，但不能造成根页面横向溢出。

## 8. 生产与手机远程访问说明

阶段 2B 只让架构兼容后续同域部署，并没有完成公网部署。生产环境应由 Nginx 或 Caddy 在同一 HTTPS 域名终止 TLS，提供前端静态文件并反向代理 `/api`；PostgreSQL、Vite 和 Uvicorn 内部端口不得公开。

默认服务只监听 `127.0.0.1`，手机无法直接访问。改为局域网或公网监听会扩大攻击面，需要单独配置 HTTPS、防火墙、反向代理、精确 Origin、密钥管理和更新策略。不要把开发服务器直接暴露到互联网，也不要把“局域网可访问”理解为“公网安全”。
