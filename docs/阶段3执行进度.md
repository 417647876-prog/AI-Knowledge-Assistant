## 状态更新规则

1. 开始 Task：总体和阶段改为 `进行中`，填写实际模型。
2. 验证失败：保持 `进行中`；无法安全继续时改为 `阻塞` 并记录原因。
3. 验证通过：勾选 Task，增加完成数，记录命令、结果和提交标识。
4. 阶段全部 Task 与质量门通过：阶段改为 `已完成`，下一阶段才能开始。
5. Terra 在同一验证上连续失败两次：停止试错，记录原因并切换 Sol。
6. 计划编写、文档确认和模型讨论不计入功能 Task 完成数。
