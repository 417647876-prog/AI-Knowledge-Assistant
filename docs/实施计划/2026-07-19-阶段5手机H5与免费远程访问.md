# 阶段 5：手机 H5 与免费远程访问实施计划

> **执行要求：** 一次只执行一个 Task。每个 Task 完成聚焦测试、差异检查和独立提交后，才开始下一个 Task。Docker、浏览器和模型重任务开始前确认物理可用内存不少于 2 GiB；不运行与当前改动无关的重复测试。

## 1. 目标与边界

阶段 5 在阶段 4 已验收的 PostgreSQL Worker、严格 owner 隔离、服务端会话/用量、额度、可观测性、管理员运营 API 和完整 Compose 之上，完成：

- 沿用现有 Vue + Element Plus 的响应式手机 H5，不创建第二套前端工程。
- 手机知识库、文档、上传进度、回收站、支持授权、流式问答、引用、服务端会话、个人用量和反馈。
- 桌面管理员用户额度、系统健康、任务、费用和质量运营元数据界面。
- 简单 `backup.ps1` / `restore.ps1`，只保护 PostgreSQL 与上传文件。
- Windows 宿主机 Tailscale Funnel 启停脚本和真实手机移动网络 HTTPS 验收。
- 阶段 5 完成后停止本轮开发，不创建阶段 6。

明确不做：公开注册、短信/微信登录、微信小程序、第二套 H5 工程、自动备份平台、云同步、正式云服务器、域名、Kubernetes 和生产级高可用。

## 2. 基线与执行门禁

- 起点分支：`codex/stage5-mobile-access`，从阶段 4 收口提交 `d27432b` 创建。
- 阶段 4 验收继续有效：后端普通回归 625 passed、PostgreSQL 集成 204 passed、前端 170 passed、Compose 20 步与重启恢复 1 passed。
- 普通业务接口继续严格按当前用户隔离；管理员前端不得新增读取普通用户正文的捷径。
- 旧 `sessionStorage` 会话不自动导入服务端，避免重复或错误归属；切换完成后只清理当前登录用户对应的旧键。
- Tailscale Funnel 的当前端口、权限和套餐规则在 Task 10 开始时重新核对官方文档，不使用设计日期的旧规则直接上线。
- 真实外网只允许 gateway；PostgreSQL、API、Worker 和内部指标端点不得发布。

- API 模块只负责 HTTP/SSE 契约；Pinia Store 持有服务端状态、分页和加载/错误状态；View 负责取数与编排；叶子组件只通过 props 读取数据并通过 emits 上报操作。
- 子组件不得自行重复请求同一资源，也不得直接修改父组件传入对象。
- View 的脚本与模板合计预计超过约 200 行时必须继续拆分，不能把原 `WorkspaceView` 替换成另一个巨型页面。
- 会话、工作区、个人页和管理员页 Store 相互独立；退出登录时按用户清理私有状态，不清理其他账号缓存。
