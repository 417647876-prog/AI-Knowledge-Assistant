# 面试演示作品后续路线图设计

## 1. 目标

将当前已完成认证与权限隔离的 AI 知识库助手，逐步完善为一套可在面试中清晰演示的 RAG 作品：既有完整用户闭环，也能用测试、评估和工程边界说明技术选择。

优先级固定为：产品闭环、检索质量、企业工程能力。

## 2. 非目标

本路线图不实现公开注册、支付、复杂组织体系、MFA、多模型 Agent 编排或完整 CI/CD 平台。它们不能直接提升当前面试演示的核心说服力。

## 3. 阶段路线

### 阶段 2C：文档管理闭环

用户可以持久化查看知识库中的文档，看到处理状态与失败原因，并执行重处理或删除。删除操作必须在同一业务事务中清理文档记录、切片、向量、任务记录和本地原文件；不存在或无权访问的资源继续返回安全的 404。

验收演示：上传文档、观察状态、重处理文档、删除文档，并确认问答不再引用已删除内容。

### 阶段 2D：问答体验完善

增加流式回答、逐步展示引用和会话内聊天历史。聊天历史只在当前浏览器会话内保存；提供新会话与清空历史操作，不在本阶段增加服务端聊天持久化。

验收演示：针对同一知识库进行多轮提问，展示回答流、引用和清空会话后的状态变化。

### 阶段 3A：检索质量

在强制 `knowledge_base_id` 过滤的前提下，将 PostgreSQL 全文检索与 pgvector 向量检索组合为混合检索，并增加可替换的 Rerank 边界。排序权重必须来自配置，检索结果保留统一的相关度和来源信息。

验收演示：对同一问题对比纯向量和混合检索的候选片段，并解释排序与引用来源。

### 阶段 3B：RAG 评估

维护小型、可提交的中文评估集，覆盖事实问答、跨段落问题、无依据拒答和引用命中。离线评估记录 Recall@K、引用命中率、拒答正确性与端到端耗时，用于比较检索策略调整前后的效果。

验收演示：运行一次评估命令，展示指标与改动前后的差异。

### 阶段 4：企业工程亮点

将 FastAPI 进程内文档处理替换为可持久化的 Worker，并补充结构化日志、基础指标、审计记录和 Docker 一键演示。任务在 API 或 Worker 重启后必须可继续被观察和重试。

验收演示：提交文档任务后重启处理进程，展示任务状态仍可查询并最终完成；根据 request_id 查询关键日志。

## 4. 依赖顺序

阶段 2C 为后续功能提供稳定的文档生命周期；2D 只依赖现有问答接口；3A 和 3B 依赖可控的文档与检索数据；阶段 4 最后替换任务执行机制，以避免过早引入运维复杂度。

```text
2C 文档管理 → 2D 问答体验 → 3A 混合检索 → 3B RAG 评估 → 4 企业工程能力
```

## 5. 阶段 2C 边界

下一阶段只处理文档列表、重处理、删除和前端状态展示。它不实现流式回答、聊天历史、混合检索、Rerank、评估平台或 Worker。

所有新增后端行为先以单元或 PostgreSQL 集成测试固定；前端操作以组件测试和浏览器验收固定。删除操作不记录或返回文件系统路径、数据库连接信息、令牌和密码。
