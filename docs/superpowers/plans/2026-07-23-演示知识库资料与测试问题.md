# 演示知识库资料与测试问题 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 生成 15 个可上传知识库的中文混合格式文件，以及 3 份共 30 道题的独立测试问题文档。

**Architecture:** 使用一个纯数据清单描述文档、事实和问题，再由可重复执行的 Python 生成器输出 Markdown、TXT、DOCX、XLSX、PDF。自动测试直接调用项目生产解析器验证所有输出可被知识库读取，并检查数量、格式、大小、文档代号、问题覆盖率和“测试题禁止上传”边界。

**Tech Stack:** Python 3.12、python-docx 1.2.0、openpyxl 3.1.5、PyMuPDF 1.28.0、pytest、项目现有 `ParserRegistry`

## Global Constraints

- 输出目录固定为项目根目录 `演示知识库资料` 和 `演示测试问题`。
- `演示知识库资料` 必须正好包含 3 个主题文件夹和 15 个知识文件。
- `.md`、`.txt`、`.docx`、`.xlsx`、`.pdf` 每种格式必须正好 3 个。
- `演示测试问题` 必须正好包含 3 个主题文件夹和 3 个 Markdown 文档，共 30 题。
- 每个知识文件必须有唯一文档代号、至少 3 个精确事实，并至少被 2 道题引用。
- 所有内容使用中文和虚构数据，不得包含真实账号、密钥、手机号、身份证号或公司机密。
- 所有知识文件必须小于 20 MB，并通过项目生产解析器读取出非空文本。
- 测试题文档必须明确写明“不要上传知识库”，避免答案污染检索。
- DOCX、XLSX、PDF 必须完成结构检查；DOCX、PDF 还必须完成可视化抽查。
- 只提交生成脚本、自动测试、15 个知识文件、3 个测试题文档和必要说明，不修改产品业务逻辑。

```python
def generate_demo_files(output_root: Path) -> tuple[Path, Path]:
    knowledge_root = output_root / "演示知识库资料"
    questions_root = output_root / "演示测试问题"
    for spec in KNOWLEDGE_DOCUMENTS:
        target = knowledge_root / spec.folder / spec.filename
        target.parent.mkdir(parents=True, exist_ok=True)
        WRITERS[target.suffix](target, spec)
    for folder, questions in QUESTION_SETS.items():
        target = questions_root / folder / "测试问题与预期答案.md"
        target.parent.mkdir(parents=True, exist_ok=True)
        write_question_set(target, folder, questions)
    return knowledge_root, questions_root
```

Renderer requirements:

- Markdown/TXT: UTF-8, title, document code, summary, full sections.
- DOCX: A4, 2.2 cm margins, Chinese title, document code, heading levels, fact summary table, footer page number.
- XLSX: `资料说明` sheet plus domain data sheet; frozen header row, filter, alternating row fill, wrapped text, widths 14–36.
- PDF: PyMuPDF with embedded `C:\Windows\Fonts\msyh.ttc` or the first available `msyh.ttc/msyh.ttf/simhei.ttf`; A4 pages, title, code, section headings, page numbers, selectable Chinese text.
- Question Markdown: warning banner, 10 numbered questions, expected points, sources, pass criteria, blank actual answer/reference fields.
- Before generation, remove only the two exact output directories under the provided `output_root`; reject `output_root` when it resolves to a filesystem root.
