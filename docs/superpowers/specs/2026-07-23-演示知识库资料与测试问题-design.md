# 演示知识库资料与测试问题设计

日期：2026-07-23  
状态：用户已批准主题、数量与混合格式方案

## 1. 目标

为 AI-Knowledge-Assistant 创建一套可直接上传的中文演示资料，用于验证：

- `.md`、`.txt`、`.docx`、`.xlsx`、`.pdf` 五种格式解析；
- 单文档检索、跨文档检索与引用来源；
- 数字、日期、规则、专有名词等精确信息召回；
- 无答案问题的边界表现；
- 项目演示时可重复执行的测试问题。

资料只使用虚构数据，不包含真实账号、密钥、客户信息或公司内部数据。

## 2. 交付目录

项目根目录新增两套互相隔离的资料。

### 2.1 可上传知识库资料

```text
演示知识库资料/
├─ 01-项目使用与故障排查/
│  ├─ 项目快速开始.md
│  ├─ 知识库上传规范.txt
│  ├─ RAG问答与引用指南.docx
│  ├─ 部署端口与健康检查.xlsx
│  └─ 常见故障排查手册.pdf
├─ 02-CSharp转AI学习笔记/
│  ├─ Python与CSharp语法对照.md
│  ├─ RAG检索流程.txt
│  ├─ FastAPI与ASP.NET-Core对照.docx
│  ├─ 向量检索参数实验.xlsx
│  └─ AI-Agent学习路线.pdf
└─ 03-模拟企业资料/
   ├─ 客服退款FAQ.md
   ├─ IT账号安全规范.txt
   ├─ 员工手册与休假制度.docx
   ├─ 产品套餐与服务时效.xlsx
   └─ 项目会议纪要与行动项.pdf
```

共 15 个文件；每个主题文件夹各包含五种受支持格式，每种格式在整套资料中出现 3 次。

### 2.2 独立测试问题

```text
演示测试问题/
├─ 01-项目使用与故障排查/
│  └─ 测试问题与预期答案.md
├─ 02-CSharp转AI学习笔记/
│  └─ 测试问题与预期答案.md
└─ 03-模拟企业资料/
   └─ 测试问题与预期答案.md
```

测试问题文档不得上传知识库，以免预期答案污染检索结果。

## 3. 内容设计

### 3.1 项目使用与故障排查

- `项目快速开始.md`：首次配置、启动、停止、访问地址和管理员创建流程。
- `知识库上传规范.txt`：支持格式、20 MB 限制、命名规范、上传后状态。
- `RAG问答与引用指南.docx`：检索、生成、引用抽屉、无答案边界和演示步骤。
- `部署端口与健康检查.xlsx`：服务、端口、健康地址、正常状态、排查命令。
- `常见故障排查手册.pdf`：Docker、内存、端口、ready 超时、解析失败等场景。

### 3.2 C# 转 AI 学习笔记

- `Python与CSharp语法对照.md`：类型、集合、异步、异常和依赖管理对照。
- `RAG检索流程.txt`：清洗、切块、向量化、召回、重排、提示词和引用。
- `FastAPI与ASP.NET-Core对照.docx`：路由、依赖注入、中间件、验证和测试。
- `向量检索参数实验.xlsx`：top-k、chunk size、overlap、MRR、延迟与实验结论。
- `AI-Agent学习路线.pdf`：工具调用、状态机、记忆、评估和阶段学习计划。

### 3.3 模拟企业资料

- `客服退款FAQ.md`：退款条件、时限、例外和升级规则。
- `IT账号安全规范.txt`：密码、MFA、离职账号、设备丢失和事件上报。
- `员工手册与休假制度.docx`：年假、病假、调休、审批和试用期规则。
- `产品套餐与服务时效.xlsx`：套餐价格、服务范围、响应时效和续费规则。
- `项目会议纪要与行动项.pdf`：虚构项目决策、负责人角色、截止日期和风险。

## 4. 可检索性规则

每个知识文件必须包含：

- 一个唯一文档代号，例如 `OPS-START-2026`；
- 至少三个可精确核对的事实，包括数字、日期、时限或状态；
- 清晰的一级、二级标题或表头；
- 800～1,800 个中文字符，表格文件按信息密度等价控制；
- 与同主题其他文件至少一个关联点，但不得出现互相矛盾的核心规则；
- 不出现真实密钥、真实手机号、真实身份证号或真实公司机密。

## 5. 测试问题设计

每个主题的 `测试问题与预期答案.md` 包含 10 题，共 30 题：

- 6 题单文档精确检索；
- 2 题跨文档综合；
- 1 题引用来源核对；
- 1 题知识库不存在答案的问题。

每题记录：

1. 问题；
2. 问题类型；
3. 预期答案要点；
4. 预期来源文件；
5. 通过标准；
6. 实际回答与实际引用的空白记录区。

每个知识文件至少被两道测试题引用一次，确保 15 个文件都能参与验收。

## 6. 文件生成与验证

- Markdown、TXT 使用 UTF-8。
- DOCX 必须能由 Word/LibreOffice 正常打开，并检查标题、表格与分页。
- XLSX 必须包含冻结表头、合理列宽和至少一个说明工作表。
- PDF 必须能正常渲染、复制文字并保持中文字体可读。
- 所有文件必须低于 20 MB。
- 对 DOCX、XLSX、PDF 执行结构检查和可视化抽查。
- 最终输出文件清单、大小、格式验证结果和建议上传顺序。

## 7. 验收标准

- `演示知识库资料` 中正好存在 3 个主题文件夹和 15 个可上传文件。
- `演示测试问题` 中正好存在 3 个主题文件夹和 3 个测试问题文档。
- 五种格式各 3 个，均可被对应解析器读取。
- 30 道测试题覆盖全部 15 个知识文件。
- 测试题文档明确标注“不要上传知识库”。
- Git 工作树仅包含本任务生成的设计、资料、测试题及必要生成脚本。
