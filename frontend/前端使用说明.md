# 阶段 4 前端与容器使用说明

## 两种本地运行方式

日常开发使用 `deploy/docker-compose.dev.yml`，只启动 PostgreSQL；前端由 Vite 的 `127.0.0.1:5173` 提供，后端由 Uvicorn 的 `127.0.0.1:8000` 提供，便于热更新。

完整演示使用 `deploy/docker-compose.yml`：gateway 在 `127.0.0.1:8080` 同源提供前端静态文件和 `/api` 代理；API、Worker、PostgreSQL 与内部指标只在容器网络中通信，不直接暴露宿主端口。

```powershell
Set-Location (git rev-parse --show-toplevel)
Copy-Item deploy/.env.example deploy/.env
notepad deploy/.env
docker compose -f deploy/docker-compose.yml up -d --build
docker compose -f deploy/docker-compose.yml ps
docker compose -f deploy/docker-compose.yml exec api python -m scripts.create_admin --username stage4-admin
```

## 健康检查与登录

- 页面：<http://127.0.0.1:8080/>
- gateway 健康：<http://127.0.0.1:8080/health>
- 后端就绪：<http://127.0.0.1:8080/api/ready>

本机 HTTP 演示应精确设置 `TRUSTED_ORIGINS=["http://127.0.0.1:8080"]`，并只在回环地址使用 `APP_ENV=development` 和 `REFRESH_COOKIE_SECURE=false`，这样 HttpOnly Refresh Cookie 才能在 gateway 重启后恢复会话。正式 HTTPS 环境必须使用 `APP_ENV=production`、`REFRESH_COOKIE_SECURE=true` 和实际 HTTPS Origin。

`JWT_SECRET_KEY`、`GATEWAY_SHARED_SECRET`、模型 API Key 与测试账号密码只放在本地环境或 `deploy/.env`，不得写入仓库、日志和验收报告。Access Token 仍只保存在前端内存，Refresh Token 仍只由浏览器通过 HttpOnly Cookie 发送。

`create_admin` 会在容器终端中安全提示输入密码；不要把密码直接写进 Compose 命令或文档。

## 停止与数据卷

普通停止保留数据：

```powershell
Set-Location (git rev-parse --show-toplevel)
docker compose -f deploy/docker-compose.yml down
```

逻辑卷包括 `knowledge_postgres_data`（数据库）、`knowledge_uploads`（上传文件）和 `knowledge_hf_cache`（模型缓存）；实际卷名会带 Compose 项目前缀。`docker compose -f deploy/docker-compose.yml down -v` 会删除这些数据，只能在明确确认后执行。

阶段 4 的命令只绑定本机回环地址。公网入口、Tailscale Funnel 和手机远程访问属于阶段 5，本说明不把它们描述成已完成能力。
